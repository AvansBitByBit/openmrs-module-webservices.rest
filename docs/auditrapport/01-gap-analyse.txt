Gap-analyse NEN-7510:2024-2

Module: openmrs-module-webservices.rest
Datum:  2026-06-19

================================================================================

A.8.3 - Toegangsbeveiliging

Status: Opgelost voor de REST-modulecode op 2026-06-19

Bewijs:
- RestUtil.ipMatches retourneert false als de allowlist leeg is.
- /session/diag vereist View RESTWS.
- Diagnostics retourneert geen rollen of privileges.

================================================================================

A.8.5 - Authenticatie

Status: Opgelost voor de REST-modulecode op 2026-06-19

Bewijs:
- AuthorizationFilter stopt foutieve Basic Auth direct met HTTP 401.
- AuthenticationRateLimiter blokkeert standaard na 5 mislukte pogingen binnen 15 minuten.
- Lockout en geweigerde lockout-verzoeken worden auditwaardig gelogd.

Resterend operationeel aandachtspunt:
- In multi-node productie is een gedeelde limiter of edge/WAF rate limiting nodig.

================================================================================

A.8.15 - Logging

Status: Opgelost voor de REST-modulecode op 2026-06-19

Bewijs:
- SecurityAuditLogger logt security-events op WARN/ERROR.
- FileAuditLogWriter schrijft auditregels append-only naar bestand.
- Auditregels redigeren wachtwoorden, tokens, cookies, BSN-patronen, diagnose en medicatie.

Resterend operationeel aandachtspunt:
- Logpad, bestandsrechten, rotatie en bewaartermijn moeten per deployment worden ingericht.

================================================================================

A.8.24 - Gebruik van cryptografie

Status: Opgelost voor de REST-modulecode op 2026-06-19

Bewijs:
- AuthorizationFilter weigert Basic Auth zonder secure request met HTTP 426 als secure transport vereist is.
- docker-compose.prod.yml zet webservices.rest.requireSecureTransport op true.
- docker-compose.dev.yml en docker-compose.test.yml zetten lokale HTTP-afwijking expliciet op false.

Resterend operationeel aandachtspunt:
- Reverse proxies moeten TLS tot aan backend borgen of de Tomcat connector correct als secure configureren.

================================================================================

A.8.28 - Veilig programmeren

Status: Opgelost voor standaard foutresponses op 2026-06-19

Bewijs:
- RestUtil.wrapErrorResponse vult code niet meer met Java class/regelnummer.
- RestUtilTest bewaakt dat class names en regelnummers niet in de standaard response verschijnen.

Resterend operationeel aandachtspunt:
- Stacktrace-details mogen alleen tijdelijk en gecontroleerd via debug-configuratie worden geactiveerd.
