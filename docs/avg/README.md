# AVG governancepakket

Dit pakket ondersteunt aantoonbaarheid onder de AVG voor deployments van de OpenMRS REST Web Services module.

De documenten zijn geen juridisch eindadvies. De verwerkingsverantwoordelijke moet organisatie-afhankelijke velden invullen, goedkeuren en actueel houden.

## Documenten

| Document | Doel |
| --- | --- |
| `verwerkingsregister.md` | Register van verwerkingen voor de REST API. |
| `dpia.md` | DPIA-template voor verwerking van gezondheidsgegevens. |
| `privacyverklaring-template.md` | Publieke informatie voor betrokkenen. |
| `verwerkersovereenkomst-checklist.md` | Contractcontrole voor hosting, beheer en supportpartijen. |
| `bewaartermijnenbeleid.md` | Bewaartermijnen en auditlog-retentie. |
| `datalekprocedure.md` | Procedure voor detectie, beoordeling en melding van datalekken. |
| `rechten-van-betrokkenen.md` | Procedure voor inzage, correctie, beperking, bezwaar en dataportabiliteit. |

## Statusvelden

Gebruik per deployment de volgende status:

- `Technisch geimplementeerd`: aantoonbaar in deze repo.
- `Operationeel te configureren`: vereist deployment- of beheerconfiguratie.
- `Organisatorisch verplicht`: vereist besluit, proces of contract buiten de code.
